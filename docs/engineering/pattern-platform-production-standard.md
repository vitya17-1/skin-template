# PatternOS: стандарт профессионального модуля выкроек

## 1. Цель

PatternOS превращает параметры изделия и подтверждённые инженерные правила в производственную геометрию.

Модуль нельзя называть профессиональным только потому, что он создаёт PDF. Профессиональный статус означает, что:

1. входные данные однозначны;
2. формулы имеют источники и область применения;
3. детали образуют собираемую конструкцию;
4. геометрия проходит автоматические проверки;
5. физический прототип изготовлен по экспортированным файлам;
6. профильный мастер подтвердил результат.

## 2. Единый pipeline

```text
Product Input
  -> Normalize
  -> Input Validation
  -> Knowledge Rules
  -> Derived Parameters
  -> Product Geometry
  -> Geometry Validation
  -> Assembly Validation
  -> Layout
  -> PDF / SVG / DXF
  -> Physical Prototype
  -> Master Approval
```

## 3. Обязательные контракты модуля

Каждый модуль изделия обязан предоставить:

- `Input Contract`: параметры пользователя, материала, фурнитуры и технологии;
- `Knowledge Contract`: источники, формулы, диапазоны и достоверность;
- `Geometry Contract`: детали, линии, отверстия, метки, припуски и сопряжённые края;
- `Validation Contract`: входные, геометрические и сборочные проверки;
- `Export Contract`: единицы, масштаб, страницы, слои и метаданные;
- `Acceptance Contract`: физический образец, измерения и заключение мастера.

## 4. Статусы готовности

### research

Собираются знания и входные данные. Производственная генерация запрещена.

### prototype-required

Геометрия работает, но не подтверждена физическим изделием.

### master-validated

Базовая конфигурация изготовлена и проверена профильным мастером. Варианты за пределами проверенного диапазона должны блокироваться.

### production-ready

Правила, диапазоны, тесты, образцы и версия методики зафиксированы. Изменение правила требует повторной приёмки.

## 5. Универсальная производственная геометрия

Общие сущности для всех изделий:

- замкнутый контур реза;
- линия шва;
- линия сгиба;
- линия утонения/спуска;
- отверстие;
- надсечка;
- контрольная точка;
- направление материала;
- зона клея;
- зона нахлёста;
- сопряжённый край;
- количество и зеркальность детали;
- допуск и технологический припуск;
- операция сборки.

Прямоугольник с подписью не является достаточной моделью детали.

## 6. Универсальные проверки

- все контуры замкнуты и не самопересекаются;
- отверстия и швы находятся в безопасной зоне;
- сопряжённые края имеют совместимую длину с учётом посадки;
- припуски не создают отрицательную или вырожденную геометрию;
- зеркальные детали действительно зеркальны;
- количество деталей согласуется со сборкой;
- материал и толщина допустимы для методики;
- детали размещаются на выбранном формате или корректно разбиваются;
- экспорт сохраняет миллиметры и масштаб 1:1;
- изменение входного параметра предсказуемо изменяет зависимую геометрию.

## 7. Приёмка физического прототипа

Для каждого релиза сохраняются:

- версия модуля и Knowledge Base;
- входные параметры;
- исходный PDF/SVG/DXF;
- фотографии раскроя и сборки;
- фактические размеры готового изделия;
- таблица отклонений;
- замечания мастера;
- решение `approved/rejected`;
- допустимый диапазон параметров, покрытый проверкой.

## 8. Ограничение ответственности системы

PatternOS обязан блокировать производственный экспорт, если обязательные данные отсутствуют или конфигурация находится вне проверенного диапазона. Предупреждение не заменяет блокировку там, где ошибка делает сборку невозможной или опасной.
