# Leather Pattern Platform MVP 0.4

Рабочий MVP платформы генерации выкроек. В 0.4 кошелек вынесен в самостоятельный product module, а PDF создается через общий engine.

## Почему MVP начинается с кошелька

Обувные выкройки требуют колодки, линии подъема, союзки, берцев, посадочных поправок и тестов на конкретной форме. Кошелек проще, но проверяет главное ядро продукта: точную параметрическую геометрию, припуски, линии сгиба, подписи деталей, контрольную линейку и экспорт PDF.

## Архитектура

- `src/components/` - UI: hero, форма, preview, мокап и скачивание PDF.
- `src/components/TemplateSelector.tsx` - готовые шаблоны кошелька.
- `src/components/ProductMockup.tsx` - схематичный вид готового изделия.
- `src/lib/patterns/` - математические генераторы выкроек в миллиметрах.
- `src/lib/pdf/` - экспорт геометрии в PDF.
- `src/data/templates/` - дефолтные шаблоны и доступные типы изделий.
- `src/types/` - типы параметров, деталей, линий и размеров.
- `src/utils/` - небольшие утилиты.
- `scripts/` - проверки геометрии.

## PDF-генерация

PDF строится через `jspdf` с единицами `mm` и форматом `a4`. SVG-preview и PDF получают одну и ту же `PatternGeometry` из `createSimpleWalletPattern`, поэтому размеры деталей не дублируются вручную.

В PDF есть:

- A4 landscape 297 x 210 мм;
- поля;
- основная деталь;
- карманы;
- линии сгиба;
- линия строчки;
- подписи;
- размеры;
- контрольная линейка 100 мм.

## Ограничения MVP

- Реализован один шаблон: простой кожаный кошелек.
- В 0.2 добавлены preset-шаблоны, но они используют один wallet pattern engine.
- Большие выкройки не разбиваются на несколько A4-листов.
- Оплата временно убрана из 0.2, чтобы быстрее тестировать PDF. Paywall можно вернуть отдельным guard перед `DownloadPanel`.
- PDF-подписи на английском, потому что стандартные шрифты jsPDF без подключения TTF надежнее работают с латиницей.

## Как добавить обувные выкройки

1. Добавить новый тип параметров в `src/types/pattern.ts`.
2. Создать генератор в `src/lib/patterns/shoe.ts`.
3. Вернуть ту же структуру `PatternGeometry`: детали, линии реза, сгиба, размеры и предупреждения.
4. Добавить шаблон в `src/data/templates/`.
5. Подключить выбор шаблона в форме.
6. При необходимости добавить tile-print для деталей больше A4.
